# Downloading Signed Documents

Once all signers have completed a document, you can download the signed PDF along with the audit trail. This guide covers manual downloads, automated retrieval, and archiving best practices.

## Download Endpoint

The download endpoint returns a short-lived, **pre-signed URL** — not the file bytes directly. Request the URL, then fetch it to retrieve the PDF.

```bash
# 1. Request a download URL (defaults to the signed document)
curl -X GET "https://app.sajn.se/api/v1/documents/doc_123/download" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```json
{
  "downloadUrl": "https://files.sajn.se/documents/doc_123/signed.pdf?signature=..."
}
```

```bash
# 2. Fetch the file from the pre-signed URL
curl -L "https://files.sajn.se/documents/doc_123/signed.pdf?signature=..." \
  -o "signed-document.pdf"
```

<Note>
  The `downloadUrl` is valid for **1 hour**. Request a fresh one if it expires.
</Note>

### Choosing which file to download

Choose which file the pre-signed URL points to with the `fileType` value:

| `fileType` | Returns | Availability |
|------------|---------|--------------|
| `SIGNED` *(default)* | The signed, sealed PDF with all signatures embedded | Document must be `COMPLETED` |
| `ORIGINAL` | The original, unsigned document | Any status |
| `JOURNAL` | The signing journal / audit trail | Document must be `COMPLETED` |

There are two forms of the endpoint:

<Tip>
  **For new integrations, prefer the path form** `GET /api/v1/documents/{id}/download/{fileType}` — the file type is explicit and required, which avoids ambiguity.
</Tip>

```bash
# Path form (recommended) — file type is part of the URL
curl -X GET "https://app.sajn.se/api/v1/documents/doc_123/download/ORIGINAL" \
  -H "Authorization: Bearer YOUR_API_KEY"

# Query form — fileType is optional and defaults to SIGNED
curl -X GET "https://app.sajn.se/api/v1/documents/doc_123/download?fileType=JOURNAL" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

Both forms return the same `{ "downloadUrl": "..." }` JSON response.

The endpoint returns `404` if the requested file doesn't exist for the document (for example, requesting `JOURNAL` before the document is completed).

### Response Headers on the pre-signed URL

When you fetch the `downloadUrl`, the file is served with:

| Header | Description |
|--------|-------------|
| `Content-Type` | `application/pdf` |
| `Content-Disposition` | `attachment; filename="document-name.pdf"` |
| `Content-Length` | File size in bytes |

## Audit Trail / Signing Journal

<Frame>
  <img src="/images/guides/audit-certificate.png" alt="Audit certificate showing signature events and verification details" />
</Frame>

The signing journal provides a detailed record of all signing events:

### Journal Contents

| Information | Description |
|-------------|-------------|
| Document ID | Unique identifier for the document |
| Document Hash | SHA-256 hash of the original document |
| Signers | List of all signers with their roles |
| Signature Events | Timestamp, IP address, signature method for each signature |
| Verification | Cryptographic proof of document integrity |

### Downloading the Journal

The signing journal is a separate file — download it with `fileType=JOURNAL`:

```bash
# 1. Get a pre-signed URL for the journal
curl -X GET "https://app.sajn.se/api/v1/documents/doc_123/download?fileType=JOURNAL" \
  -H "Authorization: Bearer YOUR_API_KEY"

# 2. Fetch it from the returned downloadUrl
curl -L "<downloadUrl from step 1>" -o "audit-journal.pdf"
```

## Storing Signed Documents

### Download to Local Storage

```javascript
const fs = require('fs');
const axios = require('axios');

async function downloadDocument(documentId, outputPath, fileType = 'SIGNED') {
  // 1. Request a pre-signed download URL
  const { data } = await axios.get(
    `https://app.sajn.se/api/v1/documents/${documentId}/download`,
    {
      headers: { 'Authorization': `Bearer ${process.env.SAJN_API_KEY}` },
      params: { fileType }
    }
  );

  // 2. Stream the file from the pre-signed URL
  const file = await axios.get(data.downloadUrl, { responseType: 'stream' });
  const writer = fs.createWriteStream(outputPath);
  file.data.pipe(writer);

  return new Promise((resolve, reject) => {
    writer.on('finish', resolve);
    writer.on('error', reject);
  });
}

// Usage
await downloadDocument('doc_123', './contracts/signed-agreement.pdf');
```

### Upload to Cloud Storage

#### Amazon S3

```javascript
const AWS = require('aws-sdk');
const axios = require('axios');

const s3 = new AWS.S3();

async function downloadAndUploadToS3(documentId, bucket, key) {
  // Get a pre-signed download URL from sajn, then fetch the file
  const { data } = await axios.get(
    `https://app.sajn.se/api/v1/documents/${documentId}/download`,
    { headers: { 'Authorization': `Bearer ${process.env.SAJN_API_KEY}` } }
  );
  const file = await axios.get(data.downloadUrl, { responseType: 'arraybuffer' });

  // Upload to S3
  await s3.putObject({
    Bucket: bucket,
    Key: key,
    Body: file.data,
    ContentType: 'application/pdf'
  }).promise();

  console.log(`Document uploaded to s3://${bucket}/${key}`);
}
```

#### Google Cloud Storage

```javascript
const { Storage } = require('@google-cloud/storage');
const axios = require('axios');

const storage = new Storage();

async function downloadAndUploadToGCS(documentId, bucket, fileName) {
  // Get a pre-signed download URL from sajn, then fetch the file
  const { data } = await axios.get(
    `https://app.sajn.se/api/v1/documents/${documentId}/download`,
    { headers: { 'Authorization': `Bearer ${process.env.SAJN_API_KEY}` } }
  );
  const download = await axios.get(data.downloadUrl, { responseType: 'arraybuffer' });

  const file = storage.bucket(bucket).file(fileName);
  await file.save(download.data, {
    contentType: 'application/pdf'
  });

  console.log(`Document uploaded to gs://${bucket}/${fileName}`);
}
```

## Automating Downloads with Webhooks

The most common pattern is to automatically download documents when they're completed using webhooks.

### Webhook Handler

```javascript
const express = require('express');
const axios = require('axios');
const crypto = require('crypto');

const app = express();
app.use(express.json());

app.post('/webhooks/sajn', async (req, res) => {
  // Verify webhook signature
  const signature = req.headers['x-sajn-signature'];
  const expectedSignature = crypto
    .createHmac('sha256', process.env.WEBHOOK_SECRET)
    .update(JSON.stringify(req.body))
    .digest('hex');

  if (signature !== expectedSignature) {
    return res.status(401).send('Invalid signature');
  }

  const { event, data } = req.body;

  // Handle document completion
  if (event === 'DOCUMENT_COMPLETED') {
    try {
      await downloadAndArchive(data.documentId, data.externalId);
      console.log(`Archived document ${data.documentId}`);
    } catch (error) {
      console.error(`Failed to archive document: ${error.message}`);
    }
  }

  res.status(200).send('OK');
});

// Fetch one file type and return its bytes
async function fetchFile(documentId, fileType) {
  const { data } = await axios.get(
    `https://app.sajn.se/api/v1/documents/${documentId}/download`,
    {
      headers: { 'Authorization': `Bearer ${process.env.SAJN_API_KEY}` },
      params: { fileType }
    }
  );
  const file = await axios.get(data.downloadUrl, { responseType: 'arraybuffer' });
  return file.data;
}

async function downloadAndArchive(documentId, externalId) {
  // Archive both the signed document and its signing journal
  const [pdf, journal] = await Promise.all([
    fetchFile(documentId, 'SIGNED'),
    fetchFile(documentId, 'JOURNAL')
  ]);

  // Store in your system (database, file system, cloud storage)
  await saveToArchive({
    documentId,
    externalId,
    pdf,
    journal,
    archivedAt: new Date()
  });
}

app.listen(3000);
```

### Python Webhook Handler

```python
from flask import Flask, request, abort
import hmac
import hashlib
import requests
import os

app = Flask(__name__)

@app.route('/webhooks/sajn', methods=['POST'])
def webhook():
    # Verify signature
    signature = request.headers.get('X-Sajn-Signature')
    body = request.get_data()
    expected = hmac.new(
        bytes(os.environ['WEBHOOK_SECRET'], 'utf-8'),
        body,
        hashlib.sha256
    ).hexdigest()

    if signature != expected:
        abort(401)

    payload = request.json
    event = payload['event']
    data = payload['data']

    if event == 'DOCUMENT_COMPLETED':
        download_and_archive(data['documentId'], data.get('externalId'))

    return 'OK', 200

def download_and_archive(document_id, external_id):
    # 1. Request a pre-signed download URL (defaults to the signed document)
    res = requests.get(
        f'https://app.sajn.se/api/v1/documents/{document_id}/download',
        headers={'Authorization': f'Bearer {os.environ["SAJN_API_KEY"]}'},
        params={'fileType': 'SIGNED'}
    )
    download_url = res.json()['downloadUrl']

    # 2. Fetch the file from the pre-signed URL
    pdf = requests.get(download_url).content

    # Save to your archive
    filename = f'contracts/{external_id or document_id}.pdf'
    with open(filename, 'wb') as f:
        f.write(pdf)

    print(f'Archived document to {filename}')
```

## Archiving Best Practices

<AccordionGroup>
  <Accordion title="Download immediately on completion">
    Set up a webhook handler to download documents as soon as they're completed. This ensures you always have a copy, even if the document is later deleted from sajn.
  </Accordion>

  <Accordion title="Download the signing journal">
    Always download the signing journal (`fileType=JOURNAL`) alongside the signed PDF for legal purposes. The journal provides proof of signing events.
  </Accordion>

  <Accordion title="Use external IDs for organization">
    Set `externalId` on documents to match your system's identifiers. This makes it easy to organize and retrieve archived documents.
  </Accordion>

  <Accordion title="Store metadata alongside PDFs">
    Save document metadata (signers, dates, custom fields) alongside the PDF for easier searching and reporting.
  </Accordion>

  <Accordion title="Implement redundant storage">
    Store signed documents in multiple locations (e.g., primary storage + backup) for disaster recovery.
  </Accordion>

  <Accordion title="Consider retention policies">
    Define how long to keep signed documents based on legal requirements. Some contracts may need to be retained for 7+ years.
  </Accordion>

  <Accordion title="Secure your archive">
    Signed documents often contain sensitive information. Ensure your archive has appropriate access controls and encryption.
  </Accordion>
</AccordionGroup>

## Error Handling

### Document Not Ready

If the requested file isn't available yet (for example, the signed PDF or journal before completion), the endpoint returns `404` with a JSON body:

```json
{
  "message": "Document is not yet fully signed. Current status: SENT",
  "code": "DOCUMENT_NOT_COMPLETED"
}
```

### Document Not Found

```json
{
  "message": "Document with ID doc_123 was not found",
  "code": "DOCUMENT_NOT_FOUND"
}
```

### Handling Errors

```javascript
async function safeDownload(documentId, fileType = 'SIGNED') {
  try {
    // Request the pre-signed URL (404 if the file isn't available yet)
    const response = await axios.get(
      `https://app.sajn.se/api/v1/documents/${documentId}/download`,
      {
        headers: { 'Authorization': `Bearer ${process.env.SAJN_API_KEY}` },
        params: { fileType },
        validateStatus: (status) => status < 500
      }
    );

    if (response.status === 404) {
      console.error(`File not available (${response.data?.code ?? 'NOT_FOUND'})`);
      return null;
    }

    // Fetch the actual file from the pre-signed URL
    const file = await axios.get(response.data.downloadUrl, { responseType: 'arraybuffer' });
    return file.data;
  } catch (error) {
    console.error(`Download failed: ${error.message}`);
    throw error;
  }
}
```
